# Contributing

Thank you for considering a contribution.

## Before opening a pull request

1. Search existing issues and pull requests.
2. Open an issue for major behavior changes before implementation.
3. Keep the core workflow local-first and usable without a paid account.
4. Do not add copyrighted lyrics, proprietary prompts, leaked credentials, or instructions that impersonate a living artist's unique voice.
5. Test the application by opening `index.html` directly in a current browser.

## Pull request checklist

- The change solves one clearly described problem.
- Existing project import/export remains compatible, or migration is documented.
- New interface text is understandable to non-technical musicians.
- No secret, API key, personal data, or tracking code is included.
- `CHANGELOG.md` is updated for user-facing changes.

## Commit style

Use concise imperative messages, for example:

- `Add version comparison summary`
- `Fix JSON import validation`
- `Document privacy model`
